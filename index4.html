Etiquetas HTML y sus Definiciones y Uso en HTML5

# Etiqueta Descripción Ejemplo de uso
67 <iframe> Representa un contexto de navegación anidado, el cual permite incrustrar otra página HTML en la página actual. Ver Ejemplo
68 <hr> Representa un cambio de tema entre párrafos (por ejemplo, un cambio de escena en una historia, un cambio de tema en una sección). Ver Ejemplo
69 <i> Muestra el texto marcado con un estilo en cursiva o italica. Ver Ejemplo
70 <input> Se usa para crear controles interactivos para formularios basados en la web con el fin de recibir datos del usuario. Ver Ejemplo
71 <ins> Marca las partes de un texto que han sido añadidos al documento. Ver Ejemplo
72 <kbd> Marca el texto que debe introducir el usuario. Ver Ejemplo
73 <label> Representa una etiqueta para un elemento en una interfaz de usuario. Ver Ejemplo
74 <legend> Crea un título para un grupos los campos <fieldset> de un formulario. Ver Ejemplo
75 <li> Declara cada uno de los elementos de una lista. Ver Ejemplo
76 <main> Representa el contenido principal del <body> de un documento o aplicación. Ver Ejemplo
77 <map> La <map> etiqueta se utiliza para definir un mapa de imágenes. Un mapa de imágenes es una imagen con áreas en las que se puede hacer clic. Ver Ejemplo
78 <menu> La <menu> etiqueta define una lista desordenada de contenido. Ver Ejemplo
79 <meter> La <meter> etiqueta define una medida escalar dentro de un rango conocido o un valor fraccionario. Esto también se conoce como calibre. Ver Ejemplo
80 <noframes> Aporta contenidos alternativos a los marcos. Las aplicaciones de usuario que no soporten marcos, o que estén configuradas para no mostrarlos, deben mostrar en su lugar el contenido de este elemento. Ver Ejemplo
81 <noscript> Aporta contenidos alternativos al elemento script. Las aplicaciones de usuario que no soporten scripts deben mostrar en su lugar el contenido de este elemento. Ver Ejemplo
82 <object> Representa un recurso externo, que puede ser tratado como una imagen, un contexto de navegación anidado, o como un recurso que debe ser manejado por un plugin. Ver Ejemplo
83 <optgroup> La <optgroup> etiqueta se utiliza para agrupar opciones relacionadas en un elemento <select> (lista desplegable). Ver Ejemplo
84 <option> Se usa para representar un item dentro de un <select>, un <optgroup> o un elemento HTML5 <datalist>. Ver Ejemplo
85 <output> La <output> etiqueta se utiliza para representar el resultado de un cálculo (como uno realizado por un script). Ver Ejemplo
86 <param> Se utiliza para definir parámetros para un elemento . Ver Ejemplo
87 <picture> Contenedor usado para especificar múltiples elementos <source> y un elemento <img> contenido en él para proveer versiones de una imagen para diferentes escenarios de dispositivos. Ver Ejemplo
88 <progress> Se utiliza para visualizar el progreso de una tarea. Ver Ejemplo
